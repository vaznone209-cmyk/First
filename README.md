# AI Vocabulary Learner - Firefox Extension (v1.1)

Расширение для изучения английских слов с помощью ИИ и экспорта в Anki.

## Что нового в версии 1.1

- **Shadow DOM**: Полная изоляция стилей тултипа от стилей сайта
- **Улучшенная обработка ошибок**: Понятные сообщения об ошибках API (401, 429, 500 и др.)
- **Safe JSON Parser**: Извлечение JSON из ответов ИИ даже с лишним текстом
- **Валидация полей**: Автоматическое заполнение отсутствующих полей

## Возможности

- Выделение текста на странице для получения перевода/определения через AI
- Всплывающее окно рядом с выделением (в стиле Yomitan)
- Генерация определений, транскрипции и примеров через LLM
- Сохранение слов в локальную коллекцию
- Экспорт коллекции в CSV формате для импорта в Anki
- Гибкая настройка API ключей и промптов
- Поддержка OpenAI, Ollama, LM Studio и других совместимых API

## Установка в Firefox

1. Откройте Firefox и перейдите на `about:debugging#/runtime/this-firefox`
2. Нажмите "Загрузить временное дополнение" (Load Temporary Add-on)
3. Выберите файл `manifest.json` из папки с расширением
4. Расширение будет установлено до перезапуска браузера

## Настройка

1. Кликните на иконку расширения в панели инструментов
2. Нажмите "Settings" чтобы открыть страницу настроек
3. Введите ваш API ключ (OpenAI, Anthropic или другой совместимый)
4. При необходимости настройте:
   - **API Endpoint**: URL для запросов к API
   - **Model**: Название модели (gpt-3.5-turbo, gpt-4, claude-3-haiku и т.д.)
   - **System Prompt**: Инструкции для ИИ о том, как объяснять слова
5. Нажмите "Test Connection" чтобы проверить подключение
6. Нажмите "Save Settings" для сохранения

## Использование

1. Выделите английский текст на любой веб-странице
2. Появится всплывающее окно с определением от ИИ
3. Нажмите "Save" чтобы добавить слово в коллекцию
4. Откройте попап расширения (клик по иконке) чтобы просмотреть сохраненные слова
5. Используйте поиск для фильтрации слов
6. Нажмите "Export" для загрузки CSV файла для Anki

## Формат экспорта для Anki

Файл CSV использует разделитель `;` и содержит поля:
- Word (Слово)
- Definition (Значение/Определение)
- Example (Пример использования)

**Импорт в Anki:**
1. Откройте Anki
2. Файл → Импорт
3. Выберите скачанный CSV файл
4. Убедитесь, что разделитель установлен как `;`
5. Выберите колоду для импорта
6. Нажмите "Import"

## Поддерживаемые API

Расширение поддерживает любые API совместимые с форматом OpenAI:

- **OpenAI API**: `https://api.openai.com/v1/chat/completions`
- **Anthropic API**: Через прокси-сервер
- **Локальные LLM**: Ollama, LM Studio, LocalAI и другие
- **Другие совместимые сервисы**: Any API с поддержкой формата chat completions

## Примеры конфигурации

### OpenAI
```
API Endpoint: https://api.openai.com/v1/chat/completions
Model: gpt-3.5-turbo
```

### Ollama (локально)
```
API Endpoint: http://localhost:11434/v1/chat/completions
Model: llama2
```

### LM Studio (локально)
```
API Endpoint: http://localhost:1234/v1/chat/completions
Model: local-model
```

## Безопасность

- API ключи хранятся только в `browser.storage.local` на вашем компьютере
- Ключи никогда не передаются третьим сторонам (кроме указанного API endpoint)
- Все запросы к API выполняются через background script расширения
- Исходный код открыт для аудита безопасности

## Требования

- Firefox 109.0 или выше
- Действующий API ключ от провайдера LLM

## Решение проблем

### Всплывающее окно не появляется
- Убедитесь, что вы выделяете текст, а не кликаете
- Проверьте, что выделение не находится в поле ввода (input/textarea)
- Попробуйте обновить страницу
- Откройте консоль браузера (F12) и проверьте наличие ошибок

### Ошибка API
- Проверьте, что API ключ введен правильно в настройках
- Убедитесь, что у вас есть баланс/лимиты на использование API
- Проверьте правильность API Endpoint и Model name
- Нажмите "Test Connection" в настройках для диагностики
- **Коды ошибок:**
  - `401`: Неверный API ключ
  - `429`: Превышен лимит запросов
  - `500/502/503`: Сервис временно недоступен
  - `Network error`: Проверьте подключение к интернету

### Слова не сохраняются
- Проверьте консоль браузера (F12) на наличие ошибок
- Убедитесь, что расширение имеет необходимые разрешения
- Попробуйте перезагрузить браузер

### Стили тултипа конфликтуют с сайтом
- Начиная с версии 1.1 используется Shadow DOM для полной изоляции
- Если проблемы сохраняются, сообщите об этом в issue

## Технические детали v1.1

### Shadow DOM
Тултип теперь создается внутри Shadow DOM (`attachShadow({mode: 'open'})`), что обеспечивает:
- Полную изоляцию CSS от стилей страницы
- Защиту от конфликтов селекторов
- Предотвращение утечек стилей наружу

### Обработка ошибок API
Все запросы обернуты в try/catch с детализацией по типам ошибок:
- HTTP статус коды (401, 403, 429, 500+)
- Сетевые ошибки (таймауты, отсутствие соединения)
- Ошибки парсинга JSON ответов

### Safe JSON Parser
Функция `safeParseJSON()` извлекает JSON из ответов ИИ:
- Прямой парсинг
- Извлечение `{...}` из текста
- Удаление markdown-блоков (```json ... ```)
- Резервное восстановление при частичных данных
